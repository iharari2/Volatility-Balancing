# Quick Reference Checklist

**Purpose:** Quick checklist for daily team coordination tasks

---

## Feature Development Checklist

### Planning Phase

- [ ] UX design review scheduled
- [ ] QA test plan created
- [ ] Documentation plan created
- [ ] User stories validated
- [ ] Acceptance criteria defined

### Development Phase

- [ ] Code review completed
- [ ] Unit tests written (>80% coverage)
- [ ] Integration tests written
- [ ] Documentation draft updated
- [ ] UX design implementation verified

### QA Phase

- [ ] Test plan executed
- [ ] All test cases passed
- [ ] Critical/high bugs fixed
- [ ] Regression tests passed
- [ ] QA sign-off obtained

### Release Phase

- [ ] UX final review completed
- [ ] Documentation published
- [ ] Release notes created
- [ ] All quality gates passed
- [ ] Stakeholder sign-off obtained

---

## Daily Standup Checklist

### Development

- [ ] What I'm working on
- [ ] Blockers identified
- [ ] Help needed from other teams

### QA

- [ ] Test execution status
- [ ] Bugs found (count and severity)
- [ ] Test coverage status

### UX

- [ ] Design reviews in progress
- [ ] UX issues identified
- [ ] User testing status

### Documentation

- [ ] Documentation status
- [ ] Review requests
- [ ] Publishing status

---

## Bug Fix Checklist

- [ ] Bug reproduced
- [ ] Root cause identified
- [ ] Fix implemented
- [ ] Tests added/updated
- [ ] QA verified fix
- [ ] Regression tests passed
- [ ] Documentation updated (if needed)
- [ ] Release approved

---

## Release Checklist

### Pre-Release (1 week before)

- [ ] All features complete
- [ ] All critical bugs fixed
- [ ] UX review completed
- [ ] Documentation published
- [ ] Release notes drafted
- [ ] Stakeholder sign-off

### Release Day

- [ ] Final regression test executed
- [ ] Production deployment plan reviewed
- [ ] Rollback plan ready
- [ ] Monitoring alerts configured
- [ ] Support team briefed

### Post-Release (1 week after)

- [ ] Error rates monitored
- [ ] User feedback collected
- [ ] QA validated production
- [ ] UX validated production
- [ ] Documentation updated
- [ ] Retrospective completed

## Quality Gate Checklist

### Gate 1: Design Review

- [ ] UX wireframes/mockups reviewed
- [ ] UX accessibility checklist reviewed
- [ ] Documentation plan created
- [ ] User stories validated
- [ ] **UX Sign-off:** [Name] - [Date]

### Gate 2: Development Checkpoint

- [ ] Code review completed
- [ ] Unit tests written (>80% coverage)
- [ ] Documentation draft updated
- [ ] UX design implementation verified
- [ ] **Dev Sign-off:** [Name] - [Date]

### Gate 3: QA Testing

- [ ] All automated tests passing
- [ ] Manual test cases executed
- [ ] Regression tests passing
- [ ] UX acceptance criteria met
- [ ] Documentation reviewed
- [ ] **QA Sign-off:** [Name] - [Date]

### Gate 4: Release Readiness

- [ ] All critical bugs fixed
- [ ] UX final review completed
- [ ] Documentation published
- [ ] Release notes prepared
- [ ] Stakeholder sign-off
- [ ] **Release Approval:** [Name] - [Date]

---

## Communication Checklist

### When Starting a Feature

- [ ] Notify QA team (test planning)
- [ ] Notify UX team (design review)
- [ ] Notify Documentation team (docs planning)
- [ ] Share feature spec/documentation

### During Development

- [ ] Update team on progress (daily standup)
- [ ] Report blockers immediately
- [ ] Request reviews when ready
- [ ] Update documentation as you code

### Before Release

- [ ] Notify all teams of release date
- [ ] Share release notes draft
- [ ] Confirm all quality gates passed
- [ ] Schedule release review meeting

---

## Metrics Tracking Checklist

### Weekly

- [ ] Test coverage metrics
- [ ] Bug metrics (found, fixed, open)
- [ ] Feature completion status
- [ ] Documentation completion status

### Monthly

- [ ] User satisfaction metrics (NPS, task completion)
- [ ] Production bug rate
- [ ] Support ticket volume
- [ ] Team velocity metrics

### Quarterly

- [ ] Process effectiveness review
- [ ] Team retrospective
- [ ] Process improvements identified
- [ ] Documentation updated

---

## Emergency Procedures Checklist

### Critical Bug in Production

- [ ] Bug identified and reported
- [ ] Dev Lead notified immediately
- [ ] Developer assigned to fix
- [ ] Fix developed and tested
- [ ] QA verified fix
- [ ] Hotfix deployed
- [ ] Documentation updated
- [ ] Post-mortem scheduled

### UX Critical Issue

- [ ] UX issue identified
- [ ] UX Lead notified
- [ ] Issue triaged (severity + priority)
- [ ] Fix planned and implemented
- [ ] UX verified fix
- [ ] Documentation updated
